# Concurrency Demo: Concurrent Processing System

Демонстрационная программа на Go, иллюстрирующая принципы параллельной обработки данных с использованием горутин, каналов и контекстов.

## Описание

Программа моделирует систему параллельной обработки, включающую две независимые рабочие подсистемы:
- **Система добычи** - обработка операций добычи ресурсов
- **Система доставки** - обработка операций доставки сообщений

Обе подсистемы работают одновременно и управляются централизованной системой контроля с временными ограничениями.

## Функциональность

### Процесс выполнения

1. **Инициализация подсистем**
   - Запуск 2 горутин для подсистемы добычи
   - Запуск 2 горутин для подсистемы доставки
   - Инициализация каналов передачи данных

2. **Параллельная обработка**
   - Каждая горутина работает независимо
   - Все горутины выполняются одновременно
   - Взаимодействие через системы передачи данных (каналы)

3. **Управление жизненным циклом**
   - Подсистема добычи работает 3 секунды
   - Подсистема доставки работает 6 секунд
   - По истечении времени горутины получают сигнал завершения

4. **Агрегирование результатов**
   - Сбор данных из всех рабочих процессов
   - Подсчёт общего количества обработанных элементов
   - Измерение времени выполнения

### Пример выполнения

```
Я шахтёр номер: 1 Начал добывать уголь!
Я почтальон номер: 1 взял письмо: Семейный привет
```

## Архитектура компонентов

### main.go
- Инициализация подсистем добычи и доставки
- Управление контекстами для синхронизации горутин
- Агрегирование результатов и вывод статистики

### miner/miner.go
- `Miner()` - рабочая функция для отдельного процесса добычи
- `MinerPool()` - инициализация пула горутин для параллельной обработки

### postman/postmna.go
- `Postman()` - рабочая функция для отдельного процесса доставки
- `PostmanPool()` - инициализация пула горутин для параллельной обработки
- `postmanToMail()` - маршрутизация данных для каждого процесса

## Ключевые концепции Go

### Горутины
Лёгкие потоки управления, запускаемые с помощью ключевого слова `go`:
```go
go functionName()
```

### Каналы
Типизированные каналы коммуникации для безопасной передачи данных между горутинами:
```go
channel := make(chan int)
channel <- value      // Отправка
value := <-channel    // Получение
```

### Контексты
Управление жизненным циклом и сигналами отмены для горутин:
```go
ctx, cancel := context.WithCancel(context.Background())
cancel() // Отправить сигнал отмены
```

### Синхронизация
Примитивы синхронизации для координации доступа к ресурсам:
```go
mtx := sync.Mutex{}
wg := &sync.WaitGroup{}
```

## Результаты выполнения

Программа выводит:
- Общее количество обработанных элементов в подсистеме добычи
- Общее количество обработанных элементов в подсистеме доставки
- Общее время выполнения программы
